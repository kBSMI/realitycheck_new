# SMI Ingestion Schema

## Overview

This document defines the wire-format schema for each of the 16 event types supported by the SMI Ingestion Simulator, the normalization mapping from raw fields to the canonical `AIWorkflowEvent` type, and the validation rules applied at the intake boundary.

---

## Canonical SMI Event Type (`AIWorkflowEvent`)

All inbound events are normalized to this shape before scoring.

```typescript
interface AIWorkflowEvent {
  id: string;                    // generated by SMI on ingest
  workflowId: string;            // maps from raw workflow_id
  baselineId: string;            // resolved from registered baselines
  timestamp: string;             // ISO 8601; from raw timestamp or ingestion time
  eventType: EventType;          // SMI event type (see mapping table)
  severity: EventSeverity;       // 'info' | 'low' | 'medium' | 'high' | 'critical'
  description: string;           // normalized human-readable description
  payload: EventPayload;         // structured fields per event type
}
```

## Normalization Mapping Summary

| Raw Field | SMI Field | Notes |
|-----------|-----------|-------|
| `workflow_id` | `workflowId` | Direct mapping |
| `timestamp` | `timestamp` | Preserved; ingestion time used if absent |
| `event_type` | `eventType` | Mapped via event type catalog |
| `violation_severity` / `severity` | `severity` | Normalized to `EventSeverity` enum |
| `guardrail_id` | `payload.guardrailId` | |
| `rule_triggered` | `payload.guardrailRule` | |
| `confidence` | `payload.behaviorConfidence` | |
| `previous_version` | `payload.expectedPolicyVersion` or `payload.previousPromptVersion` | Context-dependent |
| `new_version` | `payload.actualPolicyVersion` or `payload.newPromptVersion` | Context-dependent |
| `hash_match` | `payload.promptHashMatch` | |
| `previous_model` | `payload.previousModelVersion` | |
| `new_model` | `payload.newModelVersion` | |
| `state_key` | `payload.stateKey` | |
| `previous_state_hash` | `payload.previousStateHash` | |
| `new_state_hash` | `payload.newStateHash` | |
| `previous_orchestrator` | `payload.previousOrchestrator` | |

## Validation Rules

1. `workflow_id` — required, non-empty string
2. `event_type` — required, must match a recognized raw event type
3. `timestamp` — required, must be a valid ISO 8601 datetime string
4. `confidence` — if present, must be a number in range 0.0–1.0
5. `severity` — if present, must be one of: `info`, `low`, `medium`, `high`, `critical`
6. `raw_output_included` — must be `false` for incident records
7. `de_identified` — must be `true` for incident records
8. `metadata_only` — if `true`, raw content fields must be absent or null

Events that fail any required-field check receive `routingDecision: 'reject'` and are not scored.
